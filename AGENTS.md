# AGENTS.md

Guidance for AI coding assistants working in this repository.

## TL;DR

- `src/onepin/_cli/` and `tests/` are **hand-rolled** — edit freely.
- Everything else under `src/onepin/` is **generated by Fern** from the OpenAPI spec — do not edit; changes will be lost on the next regen.
- Lint and tests must be scoped: never run `ruff check .` or `pytest src/onepin/`.

## Repository structure

| Path | Status | Editable? |
|------|--------|-----------|
| `src/onepin/_cli/` | Hand-rolled Typer CLI | YES |
| `src/onepin/` (excluding `_cli/`) | Fern-generated SDK | NO — overwritten on every regen |
| `scripts/post_fern.sh` | Restores `py.typed` markers after regen | YES |
| `fern/` | Fern config (`fern.config.json`, `generators.yml`) for in-repo self-generation | YES |
| `src/onepin/.fernignore` | Paths Fern preserves on regen (`_cli/`, `py.typed`) | YES |
| `.github/workflows/regen.yml` | Self-generates the SDK from the OnePin API spec | YES |
| `tests/` | Project tests (`unit/`, `cli/`, `build/`) | YES |
| `src/onepin/tests/` | SDK-bundled tests | NOT COLLECTED — `testpaths = ["tests"]` |
| `src/onepin/_cli/_skill/` | Bundled cross-tool agent skill (`SKILL.md` + `reference.md`), shipped as wheel data | YES |
| `pyproject.toml`, `Makefile`, `RUNBOOK.md`, `README.md` | Project docs/config | YES |

## SDK regeneration boundary

Files under `src/onepin/` (excluding `_cli/`) are generated by [Fern](https://buildwithfern.com/) from the OnePin API OpenAPI spec, which carries the SDK shaping (`x-fern-*` + `servers`). **Do not patch them locally** — the next regen will overwrite your changes.

**If you need to change generated behavior, change the upstream OpenAPI spec.** Generation runs *in this repo*: `.github/workflows/regen.yml` fetches the shaped spec and runs `fern generate --local` (config in `fern/`), then opens a regen PR.

`src/onepin/.fernignore` lists the hand-rolled paths Fern preserves across regeneration (`_cli/`, `py.typed`). After every regen, `scripts/post_fern.sh` restores the PEP 561 markers (CI runs it automatically; local regens require running it by hand). `tests/build/test_cli_preserved.py` guards both.

If a patch on a generated file becomes unavoidable, add it to `src/onepin/.fernignore` with a comment explaining why, and document the patch in this file under a new "Active patches" section.

## CLI subpackage (`src/onepin/_cli/`)

| Module | Role |
|--------|------|
| `main.py` | Typer entry point — `onepin = "onepin._cli.main:app"` |
| `commands/` | Subcommands: `auth.py`, `workflows.py`, `voices.py`, `templates.py`, `uploads.py` |
| `commands/_registry.py` | Registers subcommands on the root `app` |
| `auth/` | Credential file (`~/.onepin/credentials`, mode `0600`) read/write |
| `_ctx.py` | Per-invocation context object |
| `_http.py` | httpx client construction (base-url + auth) |
| `_state.py` | Root-option state captured in the main callback |
| `render.py` | Rich tables / JSON output |
| `commands/skill.py` | `onepin skill install/path/uninstall` — installs the bundled agent skill into AI tools (SDK-free) |
| `_fsutil.py` | Atomic-write helper shared by `skill install` and run downloads |
| `_skill/onepin/` | The bundled `SKILL.md` + `reference.md` (package data) |

Global flags (in `main.py` callback): `--api-key` (env `ONEPIN_API_KEY`), `--base-url` (env `ONEPIN_BASE_URL`), `--workspace`, `--json`, `--no-color`, `-v/--verbose`, `--debug`.

Stack: `httpx`, `pydantic` v2, `typer`, `click`, `rich`.

## Dev commands

| Command | Purpose |
|---------|---------|
| `make install` | `uv sync --all-extras` |
| `make lint` | `uv run ruff check src/onepin/_cli tests` + `ruff format --check` (scoped — see below) |
| `make test` | `uv run pytest -q` |
| `make build` | `uv run python -m build` |
| `make publish-test` | `uv run twine upload --repository testpypi dist/*` |

CI uses scoped invocations directly:

```bash
uv run ruff check src/onepin/_cli tests
uv run ruff format --check src/onepin/_cli tests
uv run pytest -q --cov=onepin._cli --cov-report=xml --cov-fail-under=80
```

### Ruff scope quirk

`pyproject.toml` sets `tool.ruff.extend-exclude = ["src/onepin"]` so the default invocation skips the generated tree. CI then **explicitly** lints `src/onepin/_cli` because Ruff's exclude is path-prefix-only and would otherwise blanket-skip the hand-rolled subpackage too.

**Always scope lint commands to `src/onepin/_cli tests`.** Never run `ruff check .`.

## Testing

- `tests/unit/` — pure unit tests
- `tests/cli/` — Typer `CliRunner` tests for the CLI
- `tests/build/` — packaging / distribution smoke
- `respx` mocks `httpx`; `pytest-asyncio` runs in auto mode

Coverage gates: **≥80%** line on `onepin._cli`; **≥90%** diff coverage on PRs via `diff-cover`.

CI matrix: Python 3.10–3.13 × {ubuntu, macos, windows}. A `fresh-venv-smoke` job installs the built wheel and sdist into clean venvs and runs `onepin --version` to catch packaging regressions.

## Style + comment conventions

- Line length: 120 (`tool.ruff`).
- Type hints required on the public CLI surface.
- `from __future__ import annotations` at module top.

Comments must only:

1. Document a public function or class.
2. Explain the **WHY** of a non-obvious choice.
3. Explain a particularly complicated piece of code.

Comments must **never**:

- Describe what code used to do — git history is authoritative.
- Restate what the code obviously does (the WHAT).
- Reference an issue number, PR, or person — those live in the commit/PR.

## Release flow

- `release-please` opens version-bump PRs on every push to `main`.
- Merging produces an `onepin-vX.Y.Z` tag.
- `publish.yml` builds + uploads to TestPyPI, then PyPI, with attestation verification scoped to `--repo`.
- Rollback procedures: see [RUNBOOK.md](./RUNBOOK.md). Use `twine yank` (PEP 592). **Never** use `twine delete` — PyPI deletions are irrevocable.

## Common gotchas

- **Editing `src/onepin/` outside `_cli/`** → lost on next Fern regen. Change the OpenAPI spec instead.
- **`ruff check .`** → silently skips `_cli/` due to the prefix exclude. Use the scoped invocation.
- **Adding tests under `src/onepin/tests/`** → not collected. Put project tests in `tests/`.
- **Forgetting `scripts/post_fern.sh`** after a manual Fern regen → `py.typed` markers vanish; downstream type-checkers stop treating `onepin` as typed.
- **`twine delete`** on a bad release → irrevocable. Use `twine yank` (see RUNBOOK).
- **Changing the CLI surface** (e.g. adding the `skill` group) → regenerate the README block (`python scripts/gen_cli_docs.py`) and the manifest snapshot (`UPDATE_SNAPSHOT=1 pytest tests/cli/test_cli_manifest.py`), or `test_readme_in_sync.py` / `test_cli_manifest.py` fail.
- **Patching generated files locally** → the next regen wipes your fix. Either fix the spec, or add an explicit `.fernignore` entry + document the patch.
- **Bare attribute access on pydantic models** without `from __future__ import annotations` → forward refs break on older Pythons. Always import annotations at module top.

## Out of scope for in-repo work

- OpenAPI spec edits (including the SDK shaping) live upstream, not here.
- Documentation site content lives upstream.
- Credential management beyond reading/writing `~/.onepin/credentials` (mode `0600`).
